# PolicyOS

This directory contains the canonical PolicyOS layer — the cross-platform rule
system governing how all policy pillars must be designed, written, tested,
enforced, and maintained.

## Canonical documents (locked)

These documents are the authoritative PolicyOS layer.
Amendments follow the process in `policyos_governance_v1.md`.

- `policyos_platform_values_v1.md` — **Platform Values**
  The moral and political anchor for all rules. Platform values use a floor+duty
  model. Includes value-tension map, amendment protocol, fiscal authority note,
  and jurisdictional scope commentary.

- `policyos_1_0_rules_proposal.md` — **System Principles**
  Cross-platform design rules grouped into universal and targeted families,
  including KERN, ENFA, GEOG, FEDR, REGD, AIGV, ECOL, THRV, DEMO, PRIV, and ECON.

- `policyos_authoring_os_v1.md` — **Authoring OS**
  How policy must be written, tested, scoped, enforced, and maintained.
  Families include NORM, AUTH, TEST, ENFC, PLAC, and MAINT.

- `policyos_governance_v1.md` — **Governance process**
  Roles, proposal-to-adoption workflow, pillar compliance review gate,
  amendment process, and transparency requirements.

- `policyos_definitions_v1.md` — **Definitions**
  Canonical DEFN-* terms referenced throughout the PolicyOS layer set.
